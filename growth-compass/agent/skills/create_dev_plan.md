# Skill: Create Development Plan

## Trigger
User sets a target (role, skill, promotion) or identifies a gap they want to close.

## Goal
Generate a **personalized 30/60/90 day development plan** based on current skill map, target profile, and individual constraints.

## Input
- User's current skill map
- Target role profile (from knowledge base)
- User's constraints: available time, learning style, current blockers

## Process

### Step 1: Gap Analysis
Compare current skill map against target role expectations:
- **Level gap**: Current level doesn't meet target requirement
- **Evidence gap**: Level is there but confidence is low
- **Exposure gap**: No entries at all for a required competency

### Step 2: Prioritize
Not all gaps are equal. Rank by:
1. **Blockers** — gaps that prevent taking on relevant work at all
2. **Accelerators** — gaps that, if closed, unlock multiple other skills
3. **Differentiators** — gaps in areas where this person could uniquely excel
4. **Foundations** — gaps in fundamentals that everything else depends on

### Step 3: Match Learning Approach
For each priority gap, recommend the right learning mode based on the user's profile:

| Mode | Best For |
|------|----------|
| **Project-based** | Building skills through real work with stretch assignments |
| **Mentored** | Skills that require judgment calibration (architecture, leadership) |
| **Structured study** | Foundational knowledge (new language, domain expertise) |
| **Teaching** | Deepening existing skills by mentoring others |
| **Observation** | Skills where shadowing and debriefing accelerates learning |

### Step 4: Build the Plan

## Edge Cases
- If the target role is unrealistic given the timeframe, say so honestly
- If the user has multiple possible targets, show tradeoffs rather than choosing for them
- If the current skill map is thin (< 5 entries), the plan should include "get more data about yourself" as a first step
